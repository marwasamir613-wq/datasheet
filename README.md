# NOUR DATASHEET — موقع داتا شيت منتجات نور للإضاءة الحديثة

موقع/قسم احترافي لعرض منتجات **نور للإضاءة الحديثة** على شكل **داتا شيت رقمي** لكل منتج.
مبني بتقنية ويب خفيفة وسريعة (HTML / CSS / JavaScript) — بدون أي برامج أو خطوات تثبيت.

- تصميم **موبايل أولاً** ومتجاوب مع كل الشاشات
- واجهة **عربية RTL** بهوية **أزرق / أبيض**
- إحساس **ثلاثي الأبعاد** ورسوم متحركة ناعمة
- بيانات المنتجات **مأخوذة من الكتالوج الرسمي مباشرة** (٧ أقسام / ٥٨ منتج) بمقاسات دقيقة

---

## ▶️ كيف أفتح الموقع؟

**أبسط طريقة:** اضغط مرتين على ملف **`index.html`** ليفتح في المتصفح.

> ملاحظة: صور المنتجات مرتبطة من كتالوج نور على الإنترنت، لذا يفضّل وجود اتصال بالإنترنت لظهور الصور.
> أما الشعار والرسومات (مغلقة/مفتوحة/3D/المقاسات) فتظهر بدون إنترنت.

### رفع الموقع على الإنترنت (اختياري)
الموقع ملفات ثابتة، يمكن رفعه كما هو على أي استضافة مجانية:
- **Netlify** أو **Vercel**: اسحب مجلد المشروع كاملًا إلى لوحة الرفع.
- **GitHub Pages**: ارفع الملفات وفعّل Pages.
لا حاجة لأي إعداد أو خادم خاص.

## 🗂️ هيكل الملفات

```
index.html            صفحة المنتجات (الهيرو + الأقسام + كروت المنتجات + بحث وفلترة)
product.html          صفحة الداتا شيت لكل منتج (تفتح عبر ?id=...)
assets/
  css/styles.css      كل التصميم (RTL، أزرق/أبيض، 3D، متجاوب)
  js/
    data.js           بيانات كل المنتجات (مولّدة تلقائيًا — لا تُعدّل يدويًا)
    common.js          الأيقونات + أزرار التواصل + الفوتر (مشترك بين الصفحات)
    svg.js             مولّد الرسومات (مغلقة/مفتوحة/مجسم 3D/رسم المقاسات)
    app.js             منطق صفحة المنتجات
    product.js         منطق صفحة الداتا شيت
  img/
    logo.jpg           شعار نور الرسمي
    hero-panels.jpg    صورة مساندة
data/
  categories.raw.json  الأقسام كما هي من الكتالوج
  products.raw.json    المنتجات كما هي من الكتالوج (مصدر البيانات)
  build-data.js        سكربت يحوّل البيانات الخام إلى assets/js/data.js
```

## ✨ ماذا يعرض كل داتا شيت؟ (صفحة واحدة بتمرير سلس)

1. الشعار + عنوان **NOUR DATASHEET** + تعريف الشركة
2. اسم المنتج وقسمه
3. **الصورة الرئيسية** (صورة المنتج الفعلية من الكتالوج)
4. **اللوحة مغلقة** و **اللوحة مفتوحة** (رسم توضيحي بنِسب اللوحة الحقيقية)
5. **مجسم ثلاثي الأبعاد** مبني على الأبعاد الفعلية (عرض×ارتفاع×عمق)
6. **رسم المقاسات** بأسهم توضّح Width / Height / Depth
7. **جدول المقاسات** بالإنجليزية (mm و cm)
8. المميزات والمواصفات + ملاحظة **التصنيع حسب الطلب**
9. منطقة تواصل: **واتساب / اتصال / فيسبوك** + رقم الهاتف

> الترتيب: الأقسام بنفس ترتيب الكتالوج، والمنتجات داخل كل قسم **من الأصغر إلى الأكبر**.

## 🔄 تحديث البيانات لاحقًا

البيانات تُقرأ من `data/products.raw.json` و `data/categories.raw.json`.
بعد أي تعديل (أو سحب نسخة جديدة من الكتالوج) أعد توليد ملف البيانات:

```bash
node data/build-data.js
```

سيُحدّث `assets/js/data.js` تلقائيًا (يرتّب المنتجات من الأصغر للأكبر داخل كل قسم).

### تغيير بيانات التواصل أو نص التعريف
عدّل القيم في أعلى `data/build-data.js` (الهاتف، واتساب، فيسبوك، الإيميل، نص التعريف)
ثم شغّل الأمر السابق.

### استبدال الشعار
استبدل `assets/img/logo.jpg` بشعار آخر بنفس الاسم.
